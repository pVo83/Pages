# Pages — книжный магазин

Портфолио-проект многостраничного SPA для книжного магазина. Проект демонстрирует работу с Vue 3, клиентским роутингом, компонентной архитектурой, адаптивной SCSS-вёрсткой по BEM и переиспользуемой бизнес-логикой через composables.

Сайт имитирует структуру коммерческого проекта: главная страница, информация о магазине, каталог книг, статьи, контакты, доставка, политика конфиденциальности и страница 404. В проекте реализованы фильтрация и пагинация каталога, мобильное меню, формы обратной связи и подписки с клиентской валидацией.

## Стек

- **Vue 3** — интерфейс на Composition API и `<script setup>`
- **Vue Router 4** — клиентская навигация и обработка страницы 404
- **SCSS** — scoped-стили, BEM-именование, переменные и адаптивные миксины
- **Vite** — локальная разработка и production-сборка
- **Axios** — отправка формы на mock API
- **GitHub Actions** — автоматический деплой на GitHub Pages
- **ESLint и Stylelint** — проверка JavaScript, Vue-компонентов и SCSS

## Страницы

| Маршрут          | Описание                                            |
| ---------------- | --------------------------------------------------- |
| `/#/`            | Главная страница с hero-блоком, преимуществами, отзывами и статьями |
| `/#/about`       | Информация о магазине, миссия и команда             |
| `/#/articles`    | Страница со статьями и подборками                   |
| `/#/our-store`   | Каталог книг с фильтрами и пагинацией               |
| `/#/contacts-us` | Контакты, форма обратной связи, FAQ и карта         |
| `/#/shipping`    | Информация о доставке и возврате                    |
| `/#/privacy`     | Политика конфиденциальности                         |
| `*`              | Страница 404 для неизвестных маршрутов              |

Роутинг работает в hash-режиме (`/#/...`), чтобы приложение корректно открывалось на GitHub Pages.

## Возможности

- Компонентная структура страниц и повторно используемых UI-блоков
- Переиспользуемая логика в composables: `useContactForm`, `useSubscribeForm`, `useBodyLock`, `useMenuResize`, `useScrolled`, `useStoreFilters`
- Каталог книг с фильтрацией, сбросом параметров и пагинацией
- Мобильное меню с блокировкой прокрутки страницы
- Форма обратной связи с валидацией полей и состоянием `touched`
- Форма подписки с проверкой email
- Плавная прокрутка к якорям и восстановление позиции при навигации
- Адаптивная вёрстка по desktop-first подходу: 1440, 1024, 768 и 375px

## Что демонстрирует проект

- Умение собирать SPA на Vue 3 и Vue Router
- Организацию логики через composables без дублирования кода
- Работу с формами, состояниями полей и клиентской валидацией
- Поддержку адаптивной вёрстки и единого стиля классов по BEM
- Настройку сборки, линтинга и деплоя на GitHub Pages

## Запуск локально

```sh
npm install
npm run dev
```

После запуска сайт будет доступен по адресу: `http://localhost:5174/Pages/`.

## Сборка

```sh
npm run build
```

## Проверка кода

```sh
npm run lint
npm run stylelint:fix
```

## Оптимизация изображений

```sh
npm run webp
```

## Деплой

После `git push` в ветку `main` проект автоматически собирается и публикуется на GitHub Pages через GitHub Actions.

## Примечание

Форма обратной связи отправляет данные на [jsonplaceholder.typicode.com](https://jsonplaceholder.typicode.com). Это mock API, поэтому реальные данные не сохраняются.
